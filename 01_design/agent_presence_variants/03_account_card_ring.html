Alive-Presence Variant 3 — The Account-Card Ambient Ring

Each account card in the left rail carries an ambient Edge-Purple ring. The ring activates per-account when the agent is doing work for that specific account. At rest the cards look exactly as in the React preview. When Pulse is ingesting a new signal for Helix, the Helix card's ring breathes softly. When a new action is ready specifically for that account, the ring brightens and a "+1" badge appears on the card. The presence is localized — the user sees not just that the agent is working but where.

What this variant says about Pulse

Amplifies
Pulse is relational, not monolithic. The product's whole purpose is per-account understanding; the alive-presence layer mirrors that. The user can tell at a glance not just "Pulse is doing work" but "Pulse is doing work about Helix." When the burnout pattern surfaces, the user sees the cause and the destination in the same look. This is the only variant that names what the agent is working on.
Sacrifices
Highest visual complexity of the three. The left rail becomes a "weather report" that changes through the day. If 4 cards are processing at once, the rail starts to feel busy — a risk to the calm-whitespace discipline (§6 design rule 18). Mitigation: cap concurrent rings to 1–2 at a time; queue further work invisibly. Also more implementation surface in Phase 4 (per-card state subscription vs. one global signal).
Says about Pulse
"I'm working on your relationships — and you can see which one." Best fit if the CEO and the RM both value the dual-axis "Pulse is here AND is doing X." Worst fit if the user is bothered by motion in their peripheral vision; the rail will demand discipline in throttling.
